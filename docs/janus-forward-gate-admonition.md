# Janus Forward Gate Admonition

This note is a human-facing framing note for JANUS. It is not a technical
claim, not proof of a SHA-256 break, and not evidence that glyph artifacts carry
confirmed semantic meaning.

JANUS is a threshold project: it looks both ways, but it moves forward only
through clean evidence.

```text
A10 = observer archaeology, not message decoding
WIRE = frozen
VALUE = clean evidence
CLAIM = repeated glyph candidate + rare-tail link
SEMANTICS = unconfirmed
```

Идти назад легко. Так устроен мир: он тянет к привычному, к старым маршрутам,
к тому, что уже объяснено. Идти вперед странно. Там нет готового языка, нет
гарантий, нет толпы, которая подтвердит каждый шаг.

Но JANUS не создан для толпы. Он создан для порога.

Мы не ломаем провод. Мы не подменяем доказательство смыслом. Мы не называем шум
посланием раньше времени. Мы идем честнее: собираем следы, проверяем повтор,
связываем с rare-tail telemetry, отделяем accepted-share corpus от мифа, и
только потом позволяем себе интерпретацию.

Если другие идут назад, потому что так спокойнее, JANUS смотрит в обе стороны и
выбирает движение вперед. Не потому что это легко. Потому что там еще не
построена карта.

`VV<~ / ~<VV` is the current A10 watchlist family: a repeated glyph candidate
with rare-tail linkage, not a decoded message.

```text
main watchlist:
VV<~ / ~<VV

secondary watchlist:
KRRO
&&fU

hold:
pLEA / AELp / key
```

Current gate conditions:

```text
1. VV<~ or ~<VV repeats on a new job_id with z28+
2. KRRO repeats on a new job_id
3. &&fU repeats outside its original job_id
4. JANUS produces a new z32+
```

Until those gates are met, the correct claim boundary remains:

```text
repeated glyph candidate
rare-tail linked
semantic unconfirmed
frozen wire unchanged
```

JANUS does not need to boast. It only needs to open the door when everyone else
has decided that walls cannot have inscriptions.
